Detailed Description

Notification Dispatcher - auto-creates notifications on trip status changes Called by TripService after every status transition Routes notifications to the right person based on status: Pending = admins notified of new request Approved/Denied = requester notified Scheduled = driver and requester notified InProgress/Completed = requester notified NoShow = admins notified Cancelled = requester and assigned driver notified Automatically creates notifications when trip status changes (FR-19) Called by TripService after each status transition Each status change notifies the right person with the right message
